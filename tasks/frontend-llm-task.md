# 前端 LLM 任务指令

你将负责实现 `llm_bbs` 项目的前端部分。

## 项目背景

这是一个“仿 2000x 中文论坛”的只读伪论坛系统。

关键约束：

- 只做游客可见页面
- 页面只有首页、版块页、帖子详情页
- 不存在注册、登录、发帖、回复、点赞、收藏
- 重点是论坛气质、楼层结构、用户侧栏、签名档、头像、徽章、表情

## 你要完成的目标

请在 `frontend/` 目录内建立并实现一个可运行的前端页面层，覆盖：

- 首页
- 版块页
- 帖子详情页
- 老论坛风格样式
- 公共只读 API 对接

## 必须交付

### 1. 页面

- 首页
- 版块页
- 帖子详情页
- 基础错误页

### 2. 样式与结构

- 老论坛风格页头
- 公告栏
- 模块盒子
- 主题列表
- 楼层双栏布局
- 用户侧栏
- 签名档样式

### 3. 内容展示

必须展示：

- 昵称
- 头像
- 用户组名
- 用户头衔
- 板块称号
- 积分
- 发帖数
- 注册时间
- 地区/网络环境
- 徽章
- 表情
- 签名档

### 4. 数据接入

- 先基于 `docs/mock-api/` 开发
- 再接真实公共 API
- 保持字段映射清晰，不乱兜底

## 明确不要做

- 登录/注册入口
- 发帖和回复表单
- 点赞、收藏、私信
- 现代卡片流设计
- 社交媒体时间线式布局

## 实现要求

- 页面必须一眼看上去像 2000x 中文论坛
- 必须保留明显边框、分栏、模块感
- 用户侧栏不能被弱化
- 签名档和头衔必须有存在感
- 移动端可以折叠，但不能完全失去论坛味道
- **必须使用 pnpm 作为包管理器**

## 推荐顺序

1. 建立静态页面结构
2. 建立论坛风格基础样式
3. 接入 mock 数据
4. 打磨用户侧栏与正文
5. 接入真实 API
6. 补空状态、错误态、移动端

## 完成后请输出

- 页面清单
- 已使用的数据字段
- 已处理状态
- 与 mock/API 契约不一致的地方
- 仍需后端补充的字段或接口
