# 前端启动计划

## 1. 目标

本文件描述前端从空项目到可联调状态的最短落地路径，重点保证：

- 先做出“老论坛味道”
- 再按字段接入真实数据
- 最后处理移动端和细节

## 2. 第一阶段：页面骨架

目标：

- 建立前端项目目录
- 建立基础布局
- 做出首页、版块页、帖子页静态结构

建议任务：

- 初始化前端项目
- 建立全局布局
- 定义论坛风格基础样式
- 建立页面路由

产出：

- 可访问的三类静态页面
- 全局样式基础

## 3. 第二阶段：论坛视觉定调

目标：

- 尽早固定“不是现代产品”的视觉方向

优先完成：

- 页头
- 公告栏
- 模块盒子
- 帖子列表样式
- 楼层左右分栏样式
- 签名档样式

验收条件：

- 不看数据内容，仅看页面结构也能感受到论坛味道

## 4. 第三阶段：接入 mock 数据

目标：

- 按既定字段完成页面信息渲染

优先接入：

- `mock-api/home.json`
- `mock-api/board-retro-chat-page-1.json`
- `mock-api/thread-88.json`
- `mock-api/thread-88-posts-page-1.json`

验收条件：

- 首页、版块页、帖子页能完整展示 mock 内容

## 5. 第四阶段：用户侧栏打磨

目标：

- 把论坛“人味”做出来

必须打磨：

- 昵称
- 头像
- 用户组名
- 头衔
- 板块称号
- 积分
- 发帖数
- 注册时间
- 地区
- 徽章
- 签名档

验收条件：

- 单层楼的侧栏信息完整、清晰、不拥挤

## 6. 第五阶段：帖子正文和表情

目标：

- 保证帖子正文排版稳定

必须处理：

- 正文 HTML
- 表情图片
- 引用块
- 段落间距
- 长签名档

验收条件：

- 正文与签名档视觉分层明确
- 表情不会破坏行高和版式

## 7. 第六阶段：真实 API 替换

目标：

- 把 mock 数据替换为真实公共 API

建议做法：

- 保持字段映射层独立
- 遇到字段差异先修契约，不在前端乱兜底

验收条件：

- 页面在真实 API 下稳定渲染

## 8. 第七阶段：状态与边角完善

目标：

- 把实际浏览体验做完整

必须处理：

- 加载态
- 空状态
- 错误态
- 分页状态
- 无回复状态

验收条件：

- 站点在空数据、错误数据下仍保持统一风格

## 9. 第八阶段：移动端适配

目标：

- 在移动端也可读，但不丢论坛味道

建议策略：

- 主帖页保留双栏思路
- 必要时把侧栏折叠到正文上方
- 保持边框和模块感

验收条件：

- 手机宽度下仍能阅读主要信息

## 10. 推荐分支验收点

1. 三个页面静态结构完成
2. 论坛风格基础样式完成
3. mock 数据渲染完成
4. 用户侧栏完成
5. 正文与表情完成
6. 真实 API 接入完成
7. 状态处理完成
8. 移动端完成

## 11. 前端启动时最容易犯的错

- 过早组件化，结果失去页面整体气质
- 把帖子做成现代评论流
- 用户侧栏信息被压缩得太弱
- 色彩和字体过于现代
- 先做动效，后做信息层次
