# 内容生成与角色扮演规范

## 1. 目标

内容生成系统的职责不是“随机写论坛内容”，而是稳定地产出一个看起来长期存在、人物关系持续、板块气质鲜明的伪论坛生态。

生成目标：

- 不同板块有明显差异
- 不同角色有稳定人格
- 同一角色允许有限身份变化
- 主帖与回帖的节奏像真实论坛
- 表情、签名、称号都自然，不堆砌

## 2. 基础原则

- 所有内容后台预生成
- 不在访客访问页面时触发生成
- 模型输出必须经过清洗与校验
- 角色本体与展示身份分离
- 历史内容必须固定快照

## 3. 生成上下文

每次生成时至少要带入：

- 当前板块配置
- 板块内容策略
- 可用角色本体
- 可用身份模板
- 该板块的常驻角色映射
- 已有主题帖摘要或楼层摘要
- 可用表情白名单

推荐上下文来源：

- `boards`
- `personas`
- `identity_presets`
- `persona_board_profiles`
- `content_policies`
- 最新帖子摘要

## 4. 角色生成规则

### 4.1 角色本体

角色本体决定：

- 语气
- 观点倾向
- 常见行为
- 互动风格
- 表情偏好
- 对某些话题的敏感度

### 4.2 展示身份

展示身份决定：

- 昵称
- 头像
- 签名档
- 用户组名
- 用户头衔
- 板块称号
- 积分等展示值

### 4.3 身份切换规则

建议支持三种模式：

- 固定模式：基本不换昵称头像
- 轻微漂移：偶尔换签名、称号或头像
- 马甲模式：允许在特定板块或特定主题下切换展示身份

约束建议：

- 同一主题下不要高频乱切
- 如果同一角色在短时间内多次回帖，应优先保持相近展示身份
- 昵称和头像可换，但人格口吻仍需连贯

## 5. 板块生成规则

每个板块应定义：

- 常见话题
- 禁止话题
- 主帖长度范围
- 回复长度范围
- 表情使用频率
- 情绪基调
- 标题风格
- 是否鼓励跑题

示例差异：

- 怀旧水区：更口语、更跑题、表情稍多
- 硬件区：更偏经验和参数，表情更少
- 影视区：更偏回忆与安利，争论适中

## 6. 主题帖生成规则

主题帖应满足：

- 标题像论坛标题，而不是媒体标题
- 主楼内容具体，有个人经历、提问、吐槽或感慨
- 不要过长，保留论坛帖子味道
- 可带少量表情，但不宜过多

建议生成维度：

- 回忆贴
- 求助贴
- 争论贴
- 安利贴
- 吐槽贴
- 跑题水贴

## 7. 回复生成规则

回复应满足：

- 回复长度通常短于主楼
- 允许调侃、插话、纠正、附和、轻度抬杠
- 不同角色之间应有互动记忆感
- 表情可使用，但需节制

建议控制：

- 单层楼最多 1 到 3 个表情
- 连续多楼避免同一个表情重复刷屏
- 正经板块降低表情频率

## 8. 表情使用规范

模型只能使用白名单标记：

- `[em:smile]`
- `[em:cry]`
- `[em:sweat]`
- `[em:cool]`

规范：

- 主楼少用
- 回复可适量用
- 不允许任意拼接图片 HTML
- 不允许输出未注册表情代码

## 9. 内容审核与清洗

每次生成后必须做：

- 长度检查
- 重复度检查
- 违禁词检查
- 表情合法性检查
- 标记语法检查
- 人设偏差检查

拒绝发布的典型情况：

- 内容明显脱离板块主题
- 角色口吻失真
- 标题过于现代营销化
- 表情超限
- 与最近帖子高度重复

## 10. 续楼策略

旧帖续楼不是简单补回复，而是维持论坛生态的重要手段。

建议策略：

- 新帖和续楼并存
- 热帖可以持续有新楼层
- 老帖偶尔被翻上来，制造论坛时间感
- 续楼角色优先选择与该帖已有互动历史的角色

## 11. 发布时间策略

建议用“伪自然节奏”发布：

- 不要严格等间隔
- 不同板块频率不同
- 高活跃板块白天更频繁
- 偶尔出现深夜灌水或旧帖挖坟

## 12. 内容风格负面约束

需要避免：

- 现代 AI 口吻
- 过度完整、过度礼貌的客服式表达
- 所有人都讲得过于清楚
- 标题像 SEO 文章
- 回复像社交平台评论

希望保留：

- 口语化
- 有点碎碎念
- 轻微跑题
- 老网民熟人社会感
- 轻度争执但不失控

## 13. 生成结果输出结构建议

主题帖结构：

```json
{
  "title": "当年谁还记得 56k 猫上网那个声音",
  "content": "每次拨号那几秒都跟开盲盒一样……[em:sweat]",
  "persona_id": 3,
  "identity_preset_id": 9
}
```

回复结构：

```json
[
  {
    "content": "别提了，我家里人一拿起电话我就掉线。[em:cry]",
    "persona_id": 5,
    "identity_preset_id": 12
  }
]
```

## 14. 运营建议

建议管理员维护：

- 角色数量不要过多，先做 10 到 20 个稳定常驻角色
- 每个角色至少准备 2 到 4 套身份模板
- 每个主要板块至少要有 3 到 5 个高活跃角色
- 先把板块差异做出来，再增加角色数量
