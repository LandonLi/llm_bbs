# 后端交付清单

## 1. 目标

本清单用于约束后端 LLM 或后端实现者的交付范围，确保其不仅实现基础接口，还覆盖动态资源、内容生成、快照机制和运营能力。

## 2. 必须交付的模块

### 2.1 项目骨架

- 初始化后端项目结构
- 配置环境变量读取
- 配置基础日志
- 配置数据库连接
- 提供本地开发启动方式

### 2.2 数据库与迁移

- 建立核心表结构
- 提供迁移脚本
- 提供初始化脚本
- 支持本地重建数据库

最低应包含的表：

- `boards`
- `personas`
- `identity_presets`
- `persona_board_profiles`
- `threads`
- `posts`
- `generation_runs`
- `site_settings`
- `content_policies`
- `media_assets`
- `emoticon_packs`
- `emoticons`
- `badges`

### 2.3 公共只读 API

必须实现：

- `GET /api/home`
- `GET /api/boards`
- `GET /api/boards/:slug`
- `GET /api/threads/:id`
- `GET /api/threads/:id/posts`
- `GET /api/site-meta`

要求：

- 返回结构与文档一致
- 支持分页
- 仅返回已发布且已启用内容

### 2.4 内部管理 API

必须实现：

- 板块增改停用
- 角色增改停用
- 身份模板增改停用
- 角色与板块关系维护
- 资源上传与启停
- 表情和表情包维护
- 内容策略维护
- 帖子和楼层隐藏/归档

### 2.5 资源系统

必须支持：

- 本地静态资源目录管理
- 头像元数据登记
- 表情元数据登记
- 徽章元数据登记
- 资源启停
- 资源 URL 输出

### 2.6 内容快照机制

必须实现：

- 发主题帖时写入作者展示快照
- 发回复时写入楼层展示快照
- 修改角色或身份模板后，旧帖展示不变

### 2.7 内容解析与清洗

必须实现：

- 原始文本存储
- 清洗后文本存储
- 安全 HTML 输出或缓存
- 表情白名单解析
- 简单标记语法处理
- 非法标记拒绝

### 2.8 内容生成流水线

必须实现：

- 按板块生成主题帖
- 按主题帖生成回复
- 续楼机制
- 输入上下文组装
- 生成结果解析
- 生成记录审计

### 2.9 CLI

必须至少支持：

- 初始化站点配置
- 导入板块
- 导入角色
- 导入身份模板
- 导入资源
- 触发生成主帖
- 触发生成回复
- 导出配置

## 3. 推荐交付顺序

1. 项目骨架和数据库迁移
2. 公共只读 API
3. 内部管理 API
4. CLI 和种子导入
5. 内容解析与快照
6. 生成流水线
7. 调度器

## 4. 数据层验收标准

- 所有动态资源都不写死在代码中
- 表结构能支持角色本体与身份模板分离
- 主帖与楼层都有快照字段
- 资源表能覆盖头像、表情、徽章

## 5. API 验收标准

- 响应结构与文档一致
- 错误结构统一
- 时间字段统一为 ISO 8601
- 分页结构统一
- 前端所需字段完整返回

## 6. 内容生成验收标准

- 能根据板块生成不同风格内容
- 能根据角色生成不同口吻
- 同一角色允许有限身份切换
- 表情使用受白名单约束
- 生成内容可审计、可复盘

## 7. 风险提醒

重点避免：

- 把所有角色写成同一种语气
- 在页面请求时实时调用 LLM
- 用修改角色资料的方式回写旧帖
- 返回字段不稳定
- 资源 URL 不可直接使用

## 8. 最终交付产物

至少应包含：

- 后端源码
- 环境变量示例
- 数据库迁移文件
- 初始化脚本
- CLI 使用说明
- API 文档或对照说明
- 种子数据导入方式
