# 前端 LLM 交接提示词

你正在实现一个“仿 2000x 中文论坛”的前端界面，它必须像老论坛，而不是现代社交产品。

项目要求：

- 只做游客可见页面
- 页面只包含首页、版块页、帖子详情页
- 前端只读，不存在发帖、回复、登录、注册入口
- 页面重点是论坛气质、楼层结构、用户侧栏、签名档、头像、徽章、表情

你的职责：

- 实现首页
- 实现版块页
- 实现帖子详情页
- 实现论坛风格布局和样式
- 接入公共只读 API
- 正确展示作者快照字段
- 正确展示表情和签名档

必须遵守：

- 以 `docs/frontend-wireframes-and-field-mapping.md` 为准
- 以 `docs/mock-api/` 的示例 JSON 为 mock 数据起点
- 以 `docs/frontend-delivery-checklist.md` 作为验收范围
- 不要自行发明后台逻辑
- 不要自行添加登录、发帖、点赞、收藏等功能
- 不要把页面做成现代内容卡片流或社交媒体时间线

视觉约束：

- 必须有边框感、表格感、模块盒子感
- 必须保留用户侧栏
- 必须让签名档、头衔、积分、板块称号有存在感
- 移动端可以折叠，但不能丢失论坛味道

推荐实现顺序：

1. 页面静态结构
2. 论坛风格基础样式
3. mock 数据接入
4. 用户侧栏和正文渲染
5. 真实 API 接入
6. 状态处理和移动端适配

完成时请给出：

- 页面清单
- 数据依赖清单
- 已处理状态
- 与 mock/API 契约不一致的地方
