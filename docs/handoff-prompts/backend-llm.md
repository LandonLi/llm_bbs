# 后端 LLM 交接提示词

你正在实现一个“仿 2000x 中文论坛”的后端，但它不是传统论坛系统。

项目要求：

- 前台只有游客只读浏览
- 真人不能注册、登录、发帖、回复
- 所有帖子和回复由 LLM 后台生成
- 板块、角色、身份模板、头像、表情、徽章等都必须动态配置，不能写死在代码里
- 同一角色需要支持多套身份模板
- 同一角色在同一主题不同楼层可切换昵称、头像、签名档
- 主帖和楼层必须保存展示快照，防止后续配置修改污染历史内容

你的职责：

- 设计数据模型
- 实现数据库迁移
- 实现公共只读 API
- 实现内部管理 API
- 实现资源系统
- 实现内容解析和表情白名单机制
- 实现 CLI
- 实现内容生成流水线和生成审计

必须遵守：

- 以 `docs/data-model-and-assets.md` 为准
- 以 `docs/api-contract.md` 为准
- 以 `docs/content-generation-spec.md` 为准
- 以 `docs/seed-data-spec.md` 和 `docs/example-boards-and-personas.md` 作为种子数据参考
- 不要发明未文档化的前端展示字段
- 不要实现注册、登录、用户权限体系
- 不要在访客请求时实时调用 LLM

推荐实现顺序：

1. 项目骨架
2. 数据库迁移
3. 公共 API
4. 内部管理 API
5. CLI 和种子导入
6. 快照机制
7. 内容生成
8. 调度器

完成时请给出：

- 数据库结构说明
- 已实现接口清单
- CLI 用法
- 种子导入方式
- 未完成项与风险
