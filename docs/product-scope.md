# 产品范围说明

## 1. 项目目标

构建一个面向游客浏览的“仿 BBS”站点，视觉与互动氛围模仿 2000x 年代中文论坛，但技术实现保持简洁，不做任何真实用户系统。

目标体验：

- 访客打开后直觉上认为这是一个真实存在的老论坛
- 帖子、回帖、签名档、头像、称号、表情都具有时代感
- 各板块之间有明确气质差异
- 常驻角色之间有持续关系和互动记忆

## 2. 目标用户

唯一真实用户角色：

- 游客访客

唯一运营角色：

- 管理员

说明：

- 游客不能登录，也不能提交内容
- 管理员不属于“论坛用户”，而是系统运营者

## 3. 首版必须实现的前台功能

- 首页
- 版块页
- 帖子详情页
- 帖子分页
- 楼层展示
- 用户侧栏展示
- 签名档展示
- 头像展示
- 表情渲染
- 公告位
- 基础统计氛围字段

## 4. 首版必须实现的后台能力

- 动态创建、编辑、停用板块
- 动态创建、编辑、停用角色
- 动态创建、编辑、停用身份模板
- 导入、启停头像资源
- 导入、启停表情资源
- 配置板块内容策略
- 配置角色行为和互动偏好
- 触发生成主题帖
- 触发生成回复
- 隐藏或归档内容

## 5. 明确不做的功能

- 注册
- 登录
- 真人发帖
- 真人回帖
- 私信
- 站内通知
- 真正的积分升级体系
- 真正的用户组权限体系
- 版主管理体系
- 举报、申诉、封禁流程
- 复杂搜索
- 实时在线聊天

## 6. 可展示但不具备业务含义的氛围字段

这些字段允许出现在页面中，但仅用于营造真实感：

- 积分
- 发帖数
- 用户组名
- 用户头衔
- 注册时间
- 来自地区
- 板块称号
- 精华数
- 在线状态
- 徽章

约束：

- 这些值不决定权限
- 这些值不影响访问和发帖能力
- 这些值只作为展示与提示词的一部分

## 7. 页面范围定义

### 7.1 首页

需要展示：

- 站点标题与副标题
- 公告
- 版块列表
- 每个版块的简介
- 每个版块最近更新的帖子
- 今日帖数、昨日帖数、总帖数、在线人数等氛围字段

### 7.2 版块页

需要展示：

- 版块名称与版块说明
- 置顶帖
- 普通主题帖列表
- 发帖角色展示
- 回复数
- 最后回复时间
- 分页

### 7.3 帖子详情页

需要展示：

- 主楼
- 楼层回复
- 每层的用户侧栏
- 每层的表情
- 每层的签名档
- 楼层号
- 页码

## 8. 交互范围定义

允许的交互：

- 切换页面
- 翻页
- 点击帖子
- 浏览帖子内容

不允许的交互：

- 发布
- 回复
- 点赞
- 收藏
- 举报
- 关注

## 9. 风格与体验方向

希望模仿：

- 老式论坛的布局结构
- 表格感与模块盒子感
- 时代感头像、称号、签名档
- 有点话痨、有点口语化的标题与回帖风格

避免出现：

- 现代社交媒体 UI
- 过度扁平化
- 复杂的动效和卡片流
- “今天的 AI 社区产品”观感

## 10. 非目标

这个项目不是：

- 通用论坛系统
- 社交平台
- 社区中台
- 现代内容管理后台
- LLM 聊天产品

它本质上是一个“以论坛壳展示 LLM 群像内容”的内容站。
