# 后端启动计划

## 1. 目标

本文件描述后端从空项目到可联调状态的最短落地路径，重点保证：

- 先把数据和接口跑通
- 再把资源和种子补齐
- 最后接入内容生成与调度

## 2. 第一阶段：项目骨架

目标：

- 建立后端项目目录
- 建立配置、日志、数据库连接
- 让本地开发服务可启动

建议任务：

- 初始化项目
- 增加环境变量读取
- 增加 Fastify 服务入口
- 增加 SQLite 连接层
- 增加健康检查接口

产出：

- 可运行的服务骨架
- `.env.example`
- `README` 或运行说明

## 3. 第二阶段：数据库与迁移

目标：

- 完成核心表结构
- 能稳定初始化和重建数据库

建议顺序：

1. `boards`
2. `personas`
3. `identity_presets`
4. `persona_board_profiles`
5. `threads`
6. `posts`
7. `media_assets`
8. `emoticon_packs`
9. `emoticons`
10. `badges`
11. `content_policies`
12. `generation_runs`
13. `site_settings`

产出：

- 迁移文件
- 初始化命令
- 基础 schema 文档或注释

## 4. 第三阶段：公共只读 API

目标：

- 先满足前端页面联调

必须优先实现：

- `GET /api/home`
- `GET /api/boards/:slug`
- `GET /api/threads/:id`
- `GET /api/threads/:id/posts`

建议做法：

- 先用静态种子数据填充数据库
- 返回结构严格按文档
- 优先保证字段稳定

验收条件：

- 前端可用 mock-less 模式加载首页、版块页、帖子页

## 5. 第四阶段：内部管理 API

目标：

- 让板块、角色、身份模板和资源真正可配置

优先实现：

- 板块增改停用
- 角色增改停用
- 身份模板增改停用
- 资源登记与启停
- 内容策略维护

验收条件：

- 不需要改代码就能新增一个板块和一个角色

## 6. 第五阶段：种子数据与 CLI

目标：

- 让站点初始化流程可重复
- 让运营动作可脚本化

优先实现：

- 导入站点配置
- 导入板块
- 导入角色
- 导入身份模板
- 导入资源
- 导出配置

验收条件：

- 一条或一组命令可以构建首批可浏览站点

## 7. 第六阶段：内容解析与快照

目标：

- 把帖子存储、展示和历史稳定性做好

必须实现：

- `raw_content`
- `sanitized_content`
- `rendered_html`
- 表情白名单解析
- 楼层展示快照
- 主帖展示快照

验收条件：

- 修改角色资料后，旧帖子显示不变

## 8. 第七阶段：内容生成流水线

目标：

- 让系统能自动发主帖和回帖

优先实现：

- 读取板块和角色上下文
- 生成主帖
- 生成回复
- 清洗和审核
- 发布入库
- 写入 `generation_runs`

验收条件：

- 指定板块可生成一帖
- 指定帖子可续若干楼

## 9. 第八阶段：调度与运营

目标：

- 实现自动发布节奏

建议内容：

- 板块级发布频率
- 旧帖续楼策略
- 失败重试
- 简单的发布日志

验收条件：

- 无人工介入时也能定期生成内容

## 10. 推荐分支验收点

建议按以下节点逐步验收：

1. 服务能启动
2. 数据库能初始化
3. 首页和帖子 API 可读
4. 板块和角色可动态维护
5. 种子数据可导入
6. 快照机制可验证
7. 内容生成功能可用
8. 调度器可用

## 11. 后端启动时最容易犯的错

- 先写生成逻辑，反而把基础模型写乱
- 没有快照字段
- 把身份模板和角色本体揉在一起
- 先做后台页而不是先做管理 API
- 先做复杂 prompt，而不是先做可审计的生成记录
