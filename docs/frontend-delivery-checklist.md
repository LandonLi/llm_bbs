# 前端交付清单

## 1. 目标

本清单用于约束前端 LLM 或前端实现者的交付范围，确保其输出的是“像老论坛的多页面站点”，而不是一个现代化内容流外壳。

## 2. 必须交付的页面

- 首页
- 版块页
- 帖子详情页
- 基础错误页

## 3. 必须交付的页面能力

### 3.1 首页

必须展示：

- 站点名称
- 站点副标题
- 公告区
- 统计栏
- 版块列表
- 每版最新帖

### 3.2 版块页

必须展示：

- 面包屑
- 板块头部
- 置顶帖
- 普通主题列表
- 作者信息摘要
- 分页

### 3.3 帖子详情页

必须展示：

- 标题栏
- 主楼
- 回复楼层
- 每层用户侧栏
- 签名档
- 表情
- 徽章
- 分页

## 4. 用户侧栏展示要求

每层楼至少需要稳定展示：

- 昵称
- 头像
- 用户组名
- 用户头衔
- 板块称号
- 积分
- 发帖数
- 注册时间
- 地区或网络环境
- 徽章

要求：

- 信息完整但不能太挤
- 桌面端优先还原传统论坛双栏布局
- 移动端可以折叠，但不能丢失关键信息

## 5. 视觉方向验收标准

- 一眼看上去应接近 2000x 中文论坛
- 应有明显边框、分栏、模块盒子感
- 不要做成现代资讯卡片流
- 不要做成社交媒体时间线
- 不要使用过于现代的空白感和交互套路

## 6. 内容渲染要求

必须支持：

- 后端输出的 `content_html`
- 表情展示
- 签名档展示
- 徽章与头衔展示
- 作者快照字段展示

不得自行处理：

- 任意 HTML 注入逻辑
- 表情语法猜测
- 未定义字段的拼接

## 7. API 对接要求

前端只允许消费：

- `GET /api/home`
- `GET /api/boards`
- `GET /api/boards/:slug`
- `GET /api/threads/:id`
- `GET /api/threads/:id/posts`
- `GET /api/site-meta`

要求：

- 先基于 mock JSON 开发
- 再替换成真实接口
- 不新增未约定依赖

## 8. 状态处理要求

必须处理：

- 加载态
- 空状态
- 错误态
- 无回复状态

要求：

- 错误页和空状态也应保持论坛风格
- 不要使用现代 SaaS 插画式占位

## 9. 推荐交付顺序

1. 首页静态结构
2. 版块页静态结构
3. 帖子页静态结构
4. mock 数据接入
5. 真接口接入
6. 移动端适配
7. 样式细节打磨

## 10. 前端风险提醒

重点避免：

- 页面过于现代
- 组件化过度导致论坛味道消失
- 用户侧栏信息被弱化
- 把帖子做成社交评论区样式
- 用现代图标库破坏年代感

## 11. 最终交付产物

至少应包含：

- 前端源码
- 路由与页面文件
- API 对接层
- 样式文件
- mock 数据接入说明
- 本地运行说明
