# 前后端 LLM 任务拆分文档

## 1. 目标

本文件用于将项目拆分给不同 LLM 并行实现，避免双方各自发明数据结构、页面结构和交互边界。

默认角色：

- 前端 LLM
- 后端 LLM

建议顺序：

1. 先由后端 LLM 冻结数据模型与 API 契约
2. 前端 LLM 按契约使用 mock 数据开发页面
3. 后端 LLM 实现真实接口与生成流程
4. 最后联调

## 2. 前端 LLM 任务书

前端 LLM 负责：

- 首页页面结构与样式
- 版块页页面结构与样式
- 帖子详情页页面结构与样式
- 老论坛风格视觉实现
- 用户侧栏展示
- 签名档、头衔、积分、用户组名、徽章、板块称号的前台展示
- 表情渲染
- 分页、空状态、错误态
- 接入公共只读 API

前端 LLM 不负责：

- 数据库设计
- 内部管理 API
- 内容生成逻辑
- 角色行为建模
- 资源上传逻辑

前端 LLM 验收标准：

- 页面视觉明确像老中文论坛
- 不做现代社交媒体风格
- 楼层侧栏信息层次清楚
- 表情、头像、签名档显示稳定
- 全部页面只依赖公共 API

## 3. 后端 LLM 任务书

后端 LLM 负责：

- 数据模型设计与迁移
- 公共只读 API
- 内部管理 API
- CLI 管理入口
- 角色、身份模板、资源系统
- 表情白名单机制
- 内容解析与清洗
- 生成任务记录
- 调度器与内容发布流程

后端 LLM 不负责：

- 视觉表现
- CSS 风格
- 页面布局美术细节

后端 LLM 验收标准：

- 板块与角色全部可配置，不写死
- 同一角色可持有多套身份模板
- 主题帖与楼层有完整展示快照
- 公共 API 字段稳定清晰
- 内容格式和表情规则有明确白名单
- 内部 API 可被 CLI 和管理页复用

## 4. 交付顺序建议

### 4.1 第一批交付

后端 LLM：

- 数据模型草案
- 公共 API mock 响应
- 内部 API 字段设计

前端 LLM：

- 页面信息架构
- 样式方向稿
- 基于 mock 数据的静态页面

### 4.2 第二批交付

后端 LLM：

- 数据库迁移
- 公共 API 实现
- 内部 API 实现
- CLI 基础命令

前端 LLM：

- 接入真实 API
- 处理分页、错误态、空态

### 4.3 第三批交付

后端 LLM：

- LLM 内容生成流水线
- 审核清洗
- 发布与续楼

前端 LLM：

- 氛围细节打磨
- 表情、徽章、签名档优化

## 5. 接口边界硬约束

- 前端不得创建未约定字段
- 后端必须提供清晰的作者快照结构
- 表情语法由后端定义，前端只按约定渲染
- 所有时间字段统一格式
- 所有可显示资源都提供可用 URL

## 6. 推荐协作产物

建议在正式编码前，先固定以下文件：

- 产品范围说明
- 数据模型与资源模型
- API 契约
- 内容生成规范

如果做得更细，还应补：

- 页面线框图
- 设计 token 草案
- 示例板块与示范角色集

## 7. 可选的第三个实现角色

如果后续希望再拆得更清晰，可以增加第三个“运营工具 LLM”：

- 只负责内部管理页和 CLI 体验
- 不碰前台视觉
- 不碰核心生成逻辑

但首版建议仍按“前端 LLM + 后端 LLM”两路推进。
